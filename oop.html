Develop a Python program and apply protected and unprotected variables within it. 


                                
    class User:
        def __init__(self, username, password):
            self.username = username  # Unprotected variable
            self._password = password  # Protected variable
                                    
        def login(self, username, password):
            if self.username == username and self._password == password:
                print("Login successful!")
            else:
                print("Invalid username or password.")
                                    
    # Create a User instance
    user = User("myusername", "mypassword")
                                    
    # Access and modify the username (unprotected)
    print(user.username)
    user.username = "newusername"
    print(user.username)
                                    
    # Access and modify the password (protected)
    print(user._password)
    user._password = "newpassword"
    print(user._password)
                                    
    # Attempt login
    user.login("myusername", "mypassword")
    user.login("myusername", "wrongpassword")

Unit 3: UML

Discuss which UML models are most applicable at different stages of the Software Development Life Cycle.

UML models are tools that help visualise and plan software systems at different stages of development. The choice of which models to use depends on the specific project and the needs of the developers and testers involved. However, the general use of UML models during the Software Development Life Cycle could be:

  1. Requirement analysis:

    2. Use Case Diagrams: These diagrams help identify and describe what the software needs to do from the users' perspective.

  2. Planning:

    3. Class Diagrams: These diagrams show the different parts of the system, like classes and their relationships, helping us plan how the system will work.
    Sequence Diagrams: These diagrams illustrate the steps and interactions between different parts of the system, helping us understand the order of actions.
    Activity Diagrams: These diagrams show the flow of activities or steps in a process, helping us visualise how the system should work.

  3. Software design:

    4. Class Diagrams: These diagrams provide more detailed information about the structure of the system, helping developers write the actual code.
    Sequence Diagrams: These diagrams help developers understand how different parts of the system should interact with each other, which is important for coding.

  4. Software development:

    5. Class Diagrams: Developers use these diagrams to guide them when writing the code for the system.

  5. Testing:

    6. Use Case Diagrams: These diagrams are useful for creating test cases to make sure the software works as intended.
    Sequence Diagrams: Testers can use these diagrams to design tests based on the expected interactions between different parts of the system.

  6. Deployment:

    7. Deployment Diagrams: These diagrams show how the software components are deployed on hardware, helping with the setup and configuration of the system.

References:

Altvater, A. (2023) What is SDLC? understand the software development life cycle, Stackify. Available at: https://stackify.com/what-is-sdlc/

Lange, C. F. J., Chaudron, M. R. V. & Muskens, J. (2006) In practice: UML software architecture and design description. IEEE Software 23(2): 40-46.

Develop the Python program to implement the class model.


                                
    class Employee:
        def __init__(self, emp_no, name, leave_balance):
            self.emp_no = emp_no
            self.name = name
            self.leave_balance = leave_balance
                            
        def book_leave(self, num_days):
            if self.leave_balance >= num_days:
                self.leave_balance -= num_days
                print(f"Successfully booked {num_days} day(s) of leave. Your updated leave balance is {self.leave_balance} day(s).")
            else:
                print("Insufficient leave balance.")
                            
    # Create Employee
    employee1 = Employee(1, "Adesola", 7)
                            
    # Employee Details
    print("Employee Details:")
    print("Employee Number:", employee1.emp_no)
    print("Name:", employee1.name)
    print("Leave Balance:", employee1.leave_balance, "day(s)")
                            
    # Book Leave
    print("\nBooking Leave:")
    num_days = int(input("Enter the number of days of leave you want to book: "))
    employee1.book_leave(num_days)

Unit 5: More on Classes

Write a Python program with polymorphism that is usable within the summative assessment for the driverless car.


                                
    class Car:
        def __init__(self, brand, model):
            self.brand = brand
            self.model = model
                            
        def info(self):
            print(f"I am a {self.brand} {self.model}.")
                            
        def get_speed(self):
            pass
                            
        def drive(self):
            print("The car is driving.")
                            
                            
    class SUV(Car):
        def __init__(self, brand, model):
            super().__init__(brand, model)
            self.speed = 0
                            
        def get_speed(self):
            return self.speed
                            
        def drive(self):
            print(f"The {self.brand} {self.model} is driving at a speed of {self.speed}mph.")
                            
        def set_speed(self, speed):
            self.speed = speed
                            
                            
    class Sedan(Car):
        def __init__(self, brand, model):
            super().__init__(brand, model)
            self.speed = 0
                            
        def get_speed(self):
            return self.speed
                            
        def drive(self):
            print(f"The {self.brand} {self.model} is moving at a speed of {self.speed}mph.")
                            
        def set_speed(self, speed):
            self.speed = speed
                            
                            
    model_X = SUV("Tesla", "Model X")
    model_3 = Sedan("Tesla", "Model 3")
                            
    model_X.set_speed(55)
    model_3.set_speed(70)
                            
    for car in (model_X, model_3):
        car.info()
        car.drive()
